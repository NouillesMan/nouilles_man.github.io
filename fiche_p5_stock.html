Contexte & Objectifs

Une organisation gère ses stocks et sa flotte de véhicules de façon partiellement manuelle, entraînant des erreurs de suivi, des ruptures critiques et une visibilité limitée. L'objectif est de centraliser la gestion complète des assets matériels dans une application web sécurisée, avec suivi en temps réel, gestion des missions et rapports automatisés.

Fonctionnalités principales

Difficultés rencontrées

  • Mise en place de l'authentification RSA/JWT depuis zéro
  • Coordination en équipe sur un dépôt partagé (conflits Git)
  • Structuration de l'API REST avec Flask blueprints
  • Synchronisation Docker + base de données au démarrage

Solutions apportées

  • Génération de paires de clés RSA via OpenSSL, middleware JWT dédié
  • Utilisation de branches Git et pull requests systématiques
  • Organisation en blueprints Flask (auth, CRUD, root)
  • Healthcheck Docker Compose pour attendre la DB

Bilan & acquis

Ce projet m'a permis de maîtriser le développement d'une API REST sécurisée avec Flask, la mise en place d'une authentification robuste par JWT avec cryptographie asymétrique (RSA), ainsi que le déploiement d'une application complète via Docker. Le travail en équipe sur GitHub m'a également permis d'acquérir de solides pratiques de gestion de version et de collaboration en mode projet.
