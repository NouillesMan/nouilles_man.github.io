Contexte & Objectifs

L'entreprise ne disposait pas d'outil centralisé pour gérer son parc informatique. Les informations sur les PC (matériel, utilisateurs, OS, statut) étaient dispersées dans des fichiers Excel, rendant les mises à jour fastidieuses et les recherches difficiles. L'objectif est de développer une application web interne permettant la gestion complète de l'inventaire informatique (CRUD), avec interface d'administration, collecte automatique des données via clé USB et gestion fine des permissions par utilisateur.

Fonctionnalités principales

  • CRUD complet des PC (ajout, modification, suppression)
  • Recherche textuelle et filtres avancés (statut, OS, marque)
  • Tableau de bord avec statistiques
  • Gestion des utilisateurs et permissions granulaires
  • Panneau admin : champs dynamiques, options déroulantes
  • Import CSV en masse depuis l'admin
  • Scripts de collecte USB (Windows PowerShell / Linux Bash)
  • Interface responsive (desktop + mobile)
  • Protection CSRF sur tous les formulaires

Difficultés rencontrées

  • Gestion des champs dynamiques (ajout/masquage sans modifier le code)
  • Import CSV avec validation et gestion des doublons
  • Collecte USB sur FAT32 (bit exécutable absent)
  • Synchronisation des schémas SQL multiples au démarrage Docker
  • Interface responsive adaptée desktop et mobile simultanément

Solutions apportées

  • Table custom_fields pour la gestion dynamique des champs
  • Validation PHP des colonnes CSV, skip des numéros de série existants
  • Documentation explicite : sudo bash ./collect_linux.sh sur FAT32
  • Script setup.sh avec healthcheck et import automatique des SQL
  • Sidebar Bootstrap offcanvas pour mobile, fixe pour desktop
